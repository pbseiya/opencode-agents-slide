# OpenCode + Oh My OpenAgent
## คู่มือเลือกใช้ Agent ให้ถูกต้อง

---

## สไลด์ 1: ทำไมต้องรู้จัก Agent?

OpenCode มี AI Agents หลายตัวช่วยเขียนโค้ด
แต่ละตัวมีความถนัดและบทบาทแตกต่างกัน

ถ้าเลือกถูกตัวที่เหมาะสมกับงาน:
- งานเสร็จเร็ว ประหยัดเวลา
- ไม่เปลืองเครดิตหรือโควต้าเรียก API
- ได้ผลลัพธ์ที่ตรงใจตั้งแต่ครั้งแรก

ถ้าเลือกผิดหรือใช้ตัวที่ไม่เหมาะสม:
- Agent ทำงานซ้ำซ้อน วนเวียนอยู่กับที่
- เสียเวลาตอบคำถามไปมาหลายรอบ
- ผลลัพธ์ออกมาไม่ตรงตามต้องการ

ดังนั้นการเข้าใจว่าควรเลือก Agent ไหนในแต่ละสถานการณ์ จึงเป็นทักษะสำคัญ

---

## สไลด์ 2: Agent มีกี่ตัว?

ระบบ Oh My OpenAgent แบ่งเป็น 3 ระดับชั้น:

1. Agents (สั่งการระดับสูง)
   - Sisyphus - ตัวเริ่มต้น รับคำสั่งตรงๆ
   - Prometheus - วางแผนงานใหญ่
   - Atlas - ลงมือทำตามแผน
   - Hephaestus - คิดลึก วิเคราะห์ซับซ้อน

2. Core Agents (แบ่งประเภทงานย่อย)
   - core_research - ค้นคว้าหาข้อมูล
   - core_explore - สำรวจ codebase
   - core_development - พัฒนาโค้ด

3. Tools (ลงมือทำจริง)
   - terminal - รันคำสั่งเทอร์มินัล
   - file - อ่านเขียนไฟล์
   - code_execution - รันโค้ดทดสอบ

เราไม่จำเป็นต้องเรียก Core หรือ Tools เอง
เพราะ Agents ระดับบนจะตัดสินใจใช้เองอัตโนมัติ

---

## สไลด์ 3: กฎทอง

หลักการสำคัญที่ต้องจำ:

Agents มีอำนาจตัดสินใจสูงสุด
Core Agents ทำหน้าที่แบ่งประเภทงาน
Tools เป็นเพียงเครื่องมือลงมือทำ

เทียบเสมือน:
- Agents = ผู้จัดการโครงการ
- Core = หัวหน้าทีมแต่ละฝ่าย
- Tools = พนักงานปฏิบัติการ

ดังนั้นอย่าเลือก Tools เอง
และอย่าพยายามบังคับ Core Agents
ให้ Agents ระดับสูงตัดสินใจสั่งการอัตโนมัติ
จะได้ประสิทธิภาพที่ดีที่สุด

---

## สไลด์ 4: Sisyphus — ตัวเริ่มต้น

Sisyphus เป็น Agent เริ่มต้นที่เราพบเมื่อเปิด OpenCode

ใช้เมื่อ:
- งานง่าย แก้ไขไม่กี่ไฟล์
- เรารู้ว่าต้องทำอะไรพอดี
- ต้องการคำตอบเร็ว ไม่ซับซ้อน

วิธีใช้:
- เปิด OpenCode TUI ก็คือ Sisyphus อยู่แล้ว
- พิมพ์คำสั่งเป็นภาษาธรรมชาติได้เลย

ทางลัดสำคัญ:
- พิมพ์ ultrawork หรือ ulw
- Sisyphus จะทำงานแบบอัตโนมัติจนเสร็จ
- ไม่ต้องคอยตอบคำถามระหว่างทาง

เหมาะกับ:
การแก้ไขเล็กน้อย แก้บักง่ายๆ หรือถามคำถามทั่วไป

---

## สไลด์ 5: Prometheus — ผู้วางแผน

Prometheus ทำหน้าที่เหมือนสถาปนิกซอฟต์แวร์

ใช้เมื่อ:
- งานใหญ่ ไม่แน่ใจขอบเขต
- ต้องการวางแผนก่อนลงมือ
- อยากให้ช่วยแบ่งงานเป็นขั้นตอน

ขั้นตอนการใช้งาน:
1. กด Tab ที่ prompt แล้วเลือก Prometheus
2. ตอบคำถาม Interview เพื่อกำหนดขอบเขตงาน
3. Prometheus วิเคราะห์และสร้างแผนงานให้
4. เราอนุมัติแผนที่ถูกต้อง
5. พิมพ์คำสั่ง slash start work
6. Prometheus จะส่งต่อแผนให้ Atlas ลงมือทำทันที

ข้อดี:
ได้แผนที่ครอบคลุม ไม่ลืม edge case
และรู้ว่าต้องทำอะไรบ้างก่อนเริ่ม

---

## สไลด์ 6: Hephaestus — ผู้คิดลึก

Hephaestus เป็น Agent ระดับเชี่ยวชาญ
ที่ชำนาญการวิเคราะห์เชิงลึก

ใช้เมื่อ:
- ติดบักที่หาสาเหตุยาก
- ต้องค้นคว้าเทคโนโลยีหรือ domain ที่ไม่คุ้นเคย
- ต้องออกแบบใหม่หรือรีแฟกเตอร์ที่กระทบหลายส่วน

สัญญาณที่บ่งบอกว่าควรเรียก Hephaestus:
- Agent ตัวอื่นวนเวียนไม่หาสาเหตุเจอ
- ต้องอ่านโค้ดข้ามหลายไฟล์เพื่อเข้าใจปัญหา
- ต้องเปรียบเทียบหลายทางเลือกก่อนตัดสินใจ

วิธีเรียกใช้:
กด Tab ที่ prompt แล้วเลือก Hephaestus
จากนั้นอธิบายปัญหาหรือคำถามที่ต้องการวิเคราะห์ลึก

---

## สไลด์ 7: ตารางตัดสินใจ

ตารางนี้สรุปว่าควรใช้ Agent ไหนในแต่ละสถานการณ์:

แก้ไขตัวอักษรผิด พิมพ์ผิด หรือแก้ไฟล์เดียว:
ใช้ Sisyphus ตรงๆ ไม่ต้องวางแผน

พัฒนาฟีเจอร์ใหม่หรืองานขนาดกลาง:
ใช้ Prometheus เพื่อวางแผน แล้วส่งให้ Atlas ทำ

รีแฟกเตอร์ระบบใหญ่หรือเปลี่ยนแปลงโครงสร้าง:
เริ่มจาก Prometheus ส่ง Atlas และปรึกษา Hephaestus

ยังไม่รู้ว่าต้องทำอะไรดี:
ให้ Prometheus สัมภาษณ์และช่วยกำหนดขอบเขต

ดีบักปัญหาซับซ้อนหรือค้นคว้าลึก:
ใช้ Hephaestus โดยเฉพาะ

ขี้เกียจอธิบายรายละเอียดมาก:
ใช้คำสั่ง ultrawork ใน Sisyphus

---

## สไลด์ 8: สิ่งไม่ควรทำ

ข้อห้ามสำคัญที่มือใหม่มักทำผิด:

อย่าเริ่มที่ Atlas โดยตรงถ้าไม่มีแผนงาน
Atlas ทำหน้าที่ลงมือทำตามแผน
ไม่ใช่วางแผนเอง ถ้าไม่มีแผนจะทำงานมั่วไปหมด

ไม่ต้องเรียก Oracle เองด้วยตรง
Oracle จะถูกเรียกโดย Sisyphus หรือ Atlas อัตโนมัติ
เมื่อต้องการค้นหาความรู้ในระบบ

อย่าพยายามเลือก Core Agents หรือ Tools เอง
เช่น ไม่ต้องบอกว่าใช้ core_research หรือ terminal
ให้ Agents ระดับบนตัดสินใจเอง

และอย่าใช้ ultrawork กับงานที่ต้องการความแม่นยำสูง
เพราะมันจะทำอัตโนมัติโดยไม่ถามยืนยัน

---

## สไลด์ 9: สรุปง่ายๆ ที่ต้องจำ

จำสั้นๆ สามข้อ:

ข้อหนึ่ง งานเล็กน้อย แก้ไขง่ายๆ:
ใช้ Sisyphus พิมพ์ตรงๆ ได้เลย

ข้อสอง งานขนาดกลาง ไม่ซับซ้อนมาก:
ใช้ Sisyphus กับคำสั่ง ultrawork
ให้มันทำอัตโนมัติจนเสร็จ

ข้อสาม งานใหญ่ ต้องวางแผน:
ใช้ Prometheus วางแผน แล้วส่งให้ Atlas ทำ

และถ้างานต้องการวิเคราะห์เชิงลึก:
ให้ Hephaestus ช่วยคิด

หลักการง่ายๆ คือ:
เลือกให้ถูกตัว งานเสร็จไว ไม่เสียเวลา
