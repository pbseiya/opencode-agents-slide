OpenCode + Oh My OpenAgent

คู่มือเลือกใช้ Agent ให้ถูกต้อง

เลื่อนด้วยลูกศร ← → หรือคลิกที่ลูกศรด้านล่าง

ทำไมต้องรู้จัก Agent?

  • OpenCode มี AI Agents หลายตัวช่วยเขียนโค้ด
  • เลือกถูก = งานเสร็จเร็ว ไม่เปลืองเครดิต
  • เลือกผิด = agent ทำงานซ้ำ เสียเวลา ตอบคำถามวนไปมา

Agent มีกี่ตัว?

ระบบแบ่งเป็น 3 ระดับ:

  1. Agents — สั่งการระดับสูง (Sisyphus, Prometheus, Atlas, Hephaestus)
  2. Core Agents — แบ่งประเภทงาน (research, explore, development)
  3. Tools — ลงมือทำจริง (terminal, file, code_execution)

กฎทอง

Agents > Core > Tools

  • อย่าเลือก Tools เอง
  • ให้ Agents ตัดสินใจสั่งการอัตโนมัติ

Sisyphus — ตัวเริ่มต้น

ใช้เมื่อ: งานง่าย แก้ไฟล์เดียว รู้ว่าต้องทำอะไร

  • เปิด OpenCode TUI = Sisyphus อยู่แล้ว
  • พิมพ์คำสั่งตรงๆ ได้เลย

ทางลัด: พิมพ์ ultrawork หรือ ulw

Prometheus — ผู้วางแผน

ใช้เมื่อ: งานใหญ่ ไม่แน่ใจ scope

  1. กด Tab ที่ prompt → เลือก Prometheus
  2. ตอบคำถาม Interview กำหนดขอบเขต
  3. อนุมัติแผนที่ agent สร้างให้
  4. พิมพ์ /start-work

Prometheus จะส่งต่อให้ Atlas ลงมือทำ

Hephaestus — ผู้คิดลึก

ใช้เมื่อ: Debug ลึก หรือ Research ข้ามหลายไฟล์

สัญญาณที่ต้องเรียก:

  • Agent ตัวอื่นติดปัญหาไม่รู้สาเหตุ
  • ต้องค้นคว้า domain ที่ไม่คุ้นเคย
  • ต้อง redesign กระทบหลายส่วน

วิธีเรียก: กด Tab → เลือก Hephaestus

ตารางตัดสินใจ

งานAgent ที่ใช้
แก้ typo / ไฟล์เดียวSisyphus
ทำ feature ใหม่Prometheus → Atlas
Refactor ใหญ่Prometheus → Atlas → Hephaestus
ไม่รู้ต้องทำอะไรPrometheus (interview)
Debug ลึก / ResearchHephaestus
ขี้เกียจอธิบายultrawork

สิ่งไม่ควรทำ

  • ❌ อย่าเริ่มที่ Atlas โดยตรง ถ้าไม่มีแผน
  • ❌ ไม่ต้องเรียก Oracle เอง — Sisyphus จะเรียกให้อัตโนมัติ
  • ❌ อย่าพยายามเลือก Core หรือ Tools เอง

สรุปง่าย ๆ

🔹 งานเล็ก → Sisyphus ตรงๆ

🔹 งานกลาง → Sisyphus + ultrawork

🔹 งานใหญ่ → Prometheus → Atlas

🔹 คิดลึก → เพิ่ม Hephaestus

เลือกให้ถูก งานเสร็จไว ไม่เสียเวลา
