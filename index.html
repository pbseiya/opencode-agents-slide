ทำไมต้องเลือก OpenCode?

ถ้าคุณเคยรู้สึกผิดหวังกับ AI Coding ที่ใช้อยู่ คุณไม่ใช่คนเดียว

Claude Code — Lock ในระบบ Anthropic ใช้ได้แค่ Claude อย่างเดียว ปรับแต่งน้อย ToS จำกัด

Codex CLI — OpenAI model อย่างเดียว เปลี่ยน model ไม่ได้ Agent จำกัด

OpenCode — ฟรี! Open source! รองรับทุก model ปรับแต่งได้ลึก Agent orchestration เต็มรูปแบบ

  1. Multi-Model Orchestration — Kimi K2.6 + GPT 5.4 + Claude ทำงานร่วมกันใน workflow เดียว
  2. 13 Specialized Agents — Agent เฉพาะทาง ทำงาน parallel execution ได้

OpenCode คืออะไร?

AI-native coding IDE บน Terminal แบบ TUI ไม่กระพริบ ไม่หน่วง

"ถ้าคุณเคยเปลี่ยนจาก Windows มาใช้ Linux และรู้สึกตื่นเต้นที่ปรับแต่งทุกอย่างได้ OpenCode คือความรู้สึกนั้น"
  • ปรับแต่ง LSP, Linter, Formatter ต่อไฟล์ได้
  • ควบคุม Model Provider, Agent Workflow ได้เต็มที่
  • ใช้ ChatGPT, Claude, Gemini, Kimi, Grok ที่ subscribe อยู่แล้ว
  • ฟรี! Open Source! ไม่มี paywall

สถาปัตยกรรม 3 ระดับ

เหมือนองค์กรที่มีพนักงานหลายระดับ:

1. Agents (ระดับผู้จัดการ)
Sisyphus, Prometheus, Atlas, Hephaestus, Oracle, Metis, Momus — คิด วางแผน สั่งการ
2. Core Agents (ระดับหัวหน้าทีม)
core_research, core_explore, core_development — แบ่งประเภทงานย่อย
3. Tools (ระดับปฏิบัติการ)
terminal, file, code_execution — ลงมือทำจริง

เราไม่ต้องเรียก Core หรือ Tools เอง — Agents ระดับสูงจะตัดสินใจให้อัตโนมัติ

Multimodal รอบด้าน

Multimodal Looker ทำได้หมด:

  • 🖼️ ภาพ — วิเคราะห์ screenshot UI แล้วบอกว่าต้องแก้ CSS ตรงไหน
  • 🔊 เสียง — ฟังแล้วเขียน transcript หรือวิเคราะห์การบรรยาย
  • 📹 วิดีโอ — ดู demo แล้วเขียนโค้ดตาม feature ที่เห็น
  • 🌐 เว็บ — วิเคราะห์ความสวยงาม UI, โครงสร้าง HTML, responsive design

ส่ง screenshot ของ error → ให้ agent แก้ไขได้ทันที
ส่งวิดีโอสอนใช้งานจาก YouTube → agent สร้างโค้ดตามที่เห็น

Code Reviewer Agent

ใช้ GPT 5.3 Codex — Fine-tune มาสำหรับ code review โดยเฉพาะ

  • ตรวจสอบ code quality และมาตรฐาน
  • หาช่องโหว่ด้าน security — SQL injection, XSS, etc.
  • หา bugs ที่ซ่อนอยู่
  • Analyze performance issues — เช่น O(n²) loop ที่ไม่จำเป็น

🔒 Read-Only Agent

ดูได้ บอกได้ แต่ไม่สามารถแก้โค้ดหรือรันคำสั่ง terminal ได้
ทำให้การ review เป็นกลาง ปลอดภัย ไม่เกิด side effect

กฎทอง

Agents > Core > Tools

  • เปรียบเสมือน ผู้จัดการโครงการ → หัวหน้าทีม → พนักงานปฏิบัติการ
  • อย่าเลือก Tools เองด้วยตรง
  • อย่าพยายามบังคับ Core Agents
  • ให้ Agents ระดับสูง ตัดสินใจสั่งการอัตโนมัติ

Sisyphus — ตัวเริ่มต้น

Model: Kimi K2.6 | ใช้เมื่อ: งานง่าย แก้ไฟล์เดียว รู้ว่าต้องทำอะไร

  • เปิด OpenCode TUI = Sisyphus อยู่แล้ว
  • พิมพ์คำสั่งภาษาธรรมชาติได้เลย

ทางลัด: ultrawork หรือ ulw

Sisyphus ทำงานอัตโนมัติจนเสร็จ ไม่ต้องคอยตอบคำถาม
เหมาะกับงานที่มั่นใจว่าจะไม่ผิดพลาดร้ายแรง

Prometheus — ผู้วางแผน

Model: Kimi K2.6 | ใช้เมื่อ: งานใหญ่ ไม่แน่ใจ scope

  1. กด Tab → เลือก Prometheus
  2. ตอบคำถาม Interview กำหนดขอบเขต
  3. Prometheus วิเคราะห์ + สร้างแผนงาน
  4. อนุมัติแผน
  5. พิมพ์ /start-work → ส่งให้ Atlas ทำ

ได้แผนที่ครอบคลุม ไม่ลืม edge case รู้ว่าต้องทำอะไรบ้างก่อนเริ่ม

Hephaestus — ผู้คิดลึก

Model: GPT 5.4 | ใช้เมื่อ: Debug ลึก / Research ข้ามหลายไฟล์

สัญญาณที่ต้องเรียก:

  • Agent ตัวอื่นวนเวียนไม่หาสาเหตุเจอ
  • ต้องค้นคว้า domain ที่ไม่คุ้นเคย (library ใหม่, architecture ซับซ้อน)
  • ต้อง redesign หรือ refactor กระทบหลายส่วน

วิธีเรียก: กด Tab → เลือก Hephaestus
