# บทพากย์ภาษาไทยฉบับละเอียดสำหรับสไลด์ทั้ง 18 หน้า

---

## สไลด์ที่ 1 — เปิดตัว

สวัสดีครับ ยินดีต้อนรับสู่คู่มือสู่โลกแห่ง AI Coding Agent ที่ทรงพลังที่สุดในตอนนี้ OpenCode ร่วมกับ Oh My OpenAgent ถ้าคุณเคยใช้ Claude Code, Codex CLI, Kimi Code หรือ Qwen Code มาก่อน วันนี้ผมจะทำให้คุณเข้าใจว่าทำไม OpenCode ถึงเป็นตัวเลือกที่เหนือกว่า และจะพาคุณไปรู้จักกับ Agent ทั้งสิบสามตัวที่จะเปลี่ยนประสบการณ์การเขียนโค้ดของคุณไปตลอดกาล ไม่ว่าคุณจะเป็นนักพัฒนาเว็บ วิศวกร AI หรือนักวิทยาศาสตร์ข้อมูล การเลือก Agent ให้ถูกตัวจะช่วยให้งานเสร็จเร็วขึ้นเป็นร้อยเท่า มาเริ่มกันเลยครับ

---

## สไลด์ที่ 2 — ทำไมต้องเลือก OpenCode

ก่อนอื่น ผมอยากถามคุณกลับว่า คุณเคยรู้สึกผิดหวังกับเครื่องมือ AI Coding ที่ใช้งานอยู่หรือไม่ ถ้าเคย คุณไม่ใช่คนเดียว เครื่องมือส่วนใหญ่ที่คุณรู้จัก Claude Code ดีแต่ถูก lock ในระบบ Anthropic ใช้ได้แค่ Claude อย่างเดียว Codex CLI ของ OpenAI ก็เช่นกัน คุณไม่สามารถเปลี่ยน model หรือปรับแต่งอะไรได้ Kimi Code และ Qwen Code ก็มีข้อจำกัดในการตั้งค่าลึก และทั้งหมดนี้คุณไม่สามารถเพิ่ม Agent เสริม หรือปรับแต่ง workflow ได้ตามใจ OpenCode แตกต่างตรงที่นี่ เขาคือเครื่องมือที่เปิดกว้าง ปรับได้ทุกอย่าง รวม model จากหลายค่าย แล้วยังให้คุณควบคุม Agent ได้อย่างเต็มที่ ใช้ Kimi สำหรับงานรวดเร็ว GPT สำหรับงานลึก Claude สำหรับงานวิเคราะห์ ทั้งหมดนี้ในเครื่องมือเดียว ไม่ต้องสลับไปมา

---

## สไลด์ที่ 3 — OpenCode เทียบชั้นคู่แข่ง

มาดูตารางเปรียบเทียบกันชัดๆ Claude Code ราคา 100 ดอลล่าร์ต่อเดือน lock model เป็น Claude อย่างเดียว ปรับแต่งน้อย และติดเรื่อง ToS ที่จำกัดการใช้งาน Codex CLI ใช้แค่ GPT model ไม่สามารถใช้ model อื่นร่วมกันได้ และ agent มีจำกัด Kimi Code จาก Moonshot ดีในการวิเคราะห์ยาว แต่ ecosystem ยังไม่สมบูรณ์ Qwen Code จาก Alibaba รองรับภาษาจีนดี แต่ multimodal ยังไม่ลึกเท่า ส่วน OpenCode ราคาฟรี open source ไม่มีค่าสมาชิก รองรับทุก model ที่คุณมี subscription อยู่ ไม่ว่าจะ ChatGPT Claude Gemini Kimi Grok ปรับแต่งได้ลึกถึงระดับ hook tool การทำงานของ agent และที่สำคัญมี plugin Oh My OpenAgent ที่เพิ่มระบบ agent orchestration ให้ทำงานเป็นทีมได้เหมือน dev team จริงๆ

---

## สไลด์ที่ 4 — จุดเด่นที่ชนะขาด

แล้วอะไรที่ทำให้ OpenCode ชนะขาด จุดแรกคือการผสาน model หลายตัวใน workflow เดียว คุณสามารถใช้ Kimi K2.6 สำหรับงานเร็ว GPT 5.4 สำหรับงานลึก Claude สำหรับงานวิเคราะห์โค้ด complex จุดที่สองคือระบบ Agent Orchestration ของ Oh My OpenAgent ที่มี Agent เฉพาะทางถึงสิบสามตัว ทำงานแบบ parallel execution ได้จุดที่สามคือ LSP และ AST-Grep ที่ทำให้ agent แก้ไขโค้ดได้แม่นยำ ไม่พิมพ์ทับผิดบรรทัดเหมือนคู่แข่ง จุดที่สี่คือระบบ Hash-Anchored Edit ที่ใช้ hash ยืนยัน content ก่อนแก้ไข ลด stale-line error ลงไปได้มาก จุดที่ห้าคือรองรับ Multimodal fully ไม่ว่าจะวิเคราะห์ภาพ วิดีโอ เสียง หรือเว็บไซต์ และจุดสุดท้ายคือคำสั่งเวทมนตร์ ultrawork ที่เปิดใช้งานทุก agent พร้อมกันแค่พิมพ์คำเดียว

---

## สไลด์ที่ 5 — OpenCode คืออะไร

OpenCode คือ AI-native coding IDE ที่รันบน terminal โดยใช้สถาปัตยกรรม TUI text user interface ที่ไม่มีการกระพริบหน้าจอ ไม่หน่วง ผู้สร้างเขาพูดไว้ดีว่า ถ้าคุณเคยเปลี่ยนจาก Windows มาใช้ Linux และรู้สึกตื่นเต้นที่สามารถปรับแต่งทุกอย่างได้ OpenCode คือความรู้สึกนั้น ในโลกที่เครื่องมือสำเร็จรูปครอบงำ OpenCode ให้คุณควบคุมทุกอย่างตั้งแต่ LSP linter formatter ไปจนถึง model provider และ agent workflow คุณสามารถใช้ ChatGPT Claude Gemini ทั้งหมดที่คุณ subscribe อยู่แล้วโดยไม่ต้องเสียเงินเพิ่ม และสิ่งที่ดีที่สุดคือ OpenCode ฟรี open source ไม่มี paywall

---

## สไลด์ที่ 6 — สถาปัตยกรรม 3 ระดับ

ระบบ Oh My OpenAgent จัดโครงสร้าง Agent ออกเป็นสามระดับชั้น คล้ายกับองค์กรที่มีพนักงานหลายระดับ ระดับแรกคือ Agents ระดับสูง ประกอบด้วย Sisyphus Prometheus Atlas Hephaestus และอื่นๆ อีก ที่สั่งการระดับสูง ระดับที่สองคือ Core Agents ที่ทำหน้าที่แบ่งประเภทงานย่อย เช่น core research สำหรับค้นคว้า core explore สำหรับสำรวจโค้ดเบส และ core development สำหรับพัฒนาโค้ด ระดับที่สามคือ Tools ซึ่งเป็นเครื่องมือลงมือทำจริง terminal file และ code execution สิ่งสำคัญคือ เราไม่จำเป็นต้องเรียกใช้ Core หรือ Tools เอง เพราะ Agents ระดับสูงจะตัดสินใจใช้งานพวกเขาเองอัตโนมัติตามความเหมาะสมครับ

---

## สไลด์ที่ 7 — Agent หลักสี่ตัว

มารู้จักกับ Agent หลักที่คุณจะใช้งานบ่อยที่สุด สี่ตัวแรกคือ Sisyphus เป็น Agent เริ่มต้นที่คุณเจอเมื่อเปิด OpenCode TUI ใช้ model Kimi K2.6 เป็นหลัก ทำหน้าที่เหมือนพนักงานอัตราจ้างที่พร้อมทำงานง่ายๆ ได้ทันที Prometheus เป็นนักวางแผน ใช้ Kimi K2.6 เช่นกัน แต่ทำหน้าที่เหมือนสถาปนิกที่ออกแบบบ้านก่อนก่อสร้าง Atlas เป็นผู้ปฏิบัติงานตามแผน ใช้ Kimi K2.6 และ Hephaestus เป็นผู้คิดลึกระดับเชี่ยวชาญ ใช้ GPT 5.4 เป็นหลัก เพราะ GPT มีจุดแข็งในการ reasoning เชิงซับซ้อน สี่ตัวนี้จะเป็นผู้จัดการโครงการและทีมหลักที่ขับเคลื่อนงานของคุณ

---

## สไลด์ที่ 8 — Agent สนับสนุน อีกสี่ตัว

ต่อมาเป็น Agent สนับสนุนที่ถูกเรียกใช้อัตโนมัติ Oracle ใช้ Kimi K2.6 ทำหน้าที่ด้าน architecture และ debugging คล้ายกับที่ปรึกษาด้านเทคนิค Metis ใช้ Kimi K2.6 เป็นที่ปรึกษาแผนงานที่ช่วย Prometheus วิเคราะห์แผน Momus ใช้ Kimi K2.6 เป็นผู้วิจารณ์และตรวจสอบความถูกต้องของแผน Explore ใช้ GPT 5.4 Nano ซึ่งเป็น model ที่เร็วและถูก ทำหน้าที่สำรวจโค้ดเบสด้วย contextual grep และ librarian ใช้ Kimi K2.5 เป็นคนค้นหาเอกสาร official docs implementation บน GitHub และแหล่งความรู้อื่นๆ ทั้งหมดนี้ไม่ต้องเรียกเอง Agent หลักจะเรียกให้อัตโนมัติตามความจำเป็น

---

## สไลด์ที่ 9 — Agent พิเศษ อีกสามตัว

นอกจากนั้นยังมี Agent พิเศษที่น่าสนใจอีกสามตัว Multimodal Looker ใช้ Kimi K2.6 เป็นผู้เชี่ยวชาญการวิเคราะห์ภาพ เสียง วิดีโอ และหน้าเว็บ ถ้าคุณต้องการให้ AI ดู screenshot ของ UI แล้วบอกว่าต้องแก้ตรงไหน หรือวิเคราะห์ video demo แล้วเขียนโค้ดตาม agent ตัวนี้จะทำงาน Sisyphus Junior ใช้ GPT 5.4 Mini เป็น Agent เบาๆ สำหรับงานง่ายที่ไม่ต้องใช้ context ใหญ่ และ code reviewer ใช้ GPT 5.3 Codex เป็นผู้เชี่ยวชาญการ code review โดยเฉพาะ สามารถตรวจสอบคุณภาพโค้ด หาช่องโหว่ด้าน security bugs และ performance issues ได้ agent นี้ถูก lock เป็น read-only คือไม่สามารถแก้ไขโค้ดหรือรันคำสั่งเทอร์มินัลได้ ทำให้การ review ปลอดภัยและเป็นกลาง

---

## สไลด์ที่ 10 — โมเดลและการจับคู่ Agent

สิ่งที่ทำให้ Oh My OpenAgent เก่งคือการจับคู่ Agent กับ model ที่เหมาะสม Sisyphus Prometheus Atlas Metis Momus ใช้ Kimi K2.6 เป็นหลัก เพราะ Kimi มี context window ใหญ่ อ่านโค้ดได้ยาว และทำงานเร็ว Hephaestus Ultrabrain ใช้ GPT 5.4 เพราะ GPT เหนือกว่าในเรื่อง reasoning เชิงลึก Oracle ก็ใช้ Kimi K2.6 เพราะต้องเข้าใจ architecture ทั้งระบบ Explore และ Sisyphus Junior ใช้ GPT 5.4 Nano กับ Mini เพราะเป็นงานสำรวจที่ต้องเร็วไม่ต้องใช้ปัญญาเยอะ Librarian ใช้ Kimi K2.5 เพราะต้องอ่านเอกสารยาว Code Reviewer ใช้ Codex เพราะถูก fine-tune มาสำหรับ code review โดยเฉพาะ การจัดสรรนี้ทำให้แต่ละ agent ทำงานได้ดีที่สุดในด้านของตัวเอง

---

## สไลด์ที่ 11 — Multimodal รอบด้าน

ตอนนี้มาพูดถึงความสามารถ multimodal ที่ OpenCode + Oh My OpenAgent ทำได้ ซึ่งคู่แข่งหลายตัวยังไม่ทำได้เต็มรูปแบบ Multimodal Looker สามารถวิเคราะห์ภาพหน้าจอ UI แล้วบอกว่าต้องแก้ไข CSS ตรงไหน สามารถฟังเสียงแล้วเขียน transcript หรือวิเคราะห์การบรรยายได้ สามารถดูวิดีโอ demo แล้วเขียนโค้ดตาม feature ที่เห็นในวิดีโอได้ และสามารถเปิดหน้าเว็บแล้ววิเคราะห์ความสวยงามของ UI โครงสร้าง HTML และ responsive design ได้ นี่หมายความว่าคุณสามารถส่ง screenshot ของ error แล้วบอกให้ agent แก้ไขได้ทันที หรือส่งวิดีโอสอนใช้งานจาก YouTube แล้วให้ agent สร้างโค้ดตามที่เห็นในวิดีโอนั้น

## สไลด์ที่ 12 — Code Reviewer Agent

Code Reviewer Agent คือหนึ่งในจุดแข็งที่ไม่มีใครเหมือน ตัวนี้ถูกออกแบบมาเพื่อ code review โดยเฉพาะ ใช้ model GPT 5.3 Codex ซึ่งถูก fine-tune มาสำหรับงาน code review โดยเฉพาะ สามารถตรวจสอบ quality ของโค้ด หาช่องโหว่ด้าน security เช่น SQL injection XSS หา bugs ที่ซ่อนอยู่ และ analyze performance issues เช่น O of n squared loop ที่ไม่จำเป็น ที่สำคัญ agent ตัวนี้ถูก lock เป็น read-only หมายความว่าเขาดูได้ บอกได้ แต่ไม่สามารถแก้ไขโค้ดหรือรันคำสั่งเทอร์มินัลได้ นี่ทำให้การ review เป็นกลาง ปลอดภัย และไม่เกิด side effect ใดๆ กับโค้ดของคุณ ใช้งานโดยเรียก agent ตัวนี้มา review pull request หรือโค้ดที่คุณเพิ่งเขียนเสร็จ

---

## สไลด์ที่ 13 — กฎทอง

ก่อนไปลง detail ของแต่ละตัว มาทำความเข้าใจกฎทองกัน ลำดับความสำคัญในการตัดสินใจคือ Agents มาก่อน Core แล้วค่อยถึง Tools เปรียบเสมือนในองค์กร ผู้จัดการโครงการคือคนตัดสินใจว่าจะส่งงานไปให้ทีมไหน หัวหน้าทีมแต่ละฝ่ายเป็นคนแบ่งงานย่อย และพนักงานปฏิบัติการเป็นคนลงมือทำจริง ถ้าคุณลูกค้าเดินเข้ามาสั่งงานตรงๆ กับพนักงานปฏิบัติการโดยไม่ผ่านหัวหน้างาน งานอาจออกมาผิดทิศทาง ดังนั้นอย่าเลือก Tools เองด้วยตรง และอย่าพยายามบังคับ Core Agents ให้ Agents ระดับสูงตัดสินใจสั่งการอัตโนมัติ ระบบจะจัดสรรทรัพยากรได้อย่างมีประสิทธิภาพที่สุด

---

## สไลด์ที่ 14 — Sisyphus ตัวเริ่มต้น

Sisyphus เป็น Agent เริ่มต้นที่คุณจะพบเมื่อเปิด OpenCode TUI ใช้ model Kimi K2.6 เป็นหลัก ทำหน้าที่เหมือนพนักงานอัตราจ้างที่พร้อมทำงานง่ายๆ ได้ทันที คุณควรใช้เมื่องานไม่ซับซ้อน เช่น แก้ไขไฟล์เดียว แก้คำผิด หรือถามคำถามที่มีคำตอบตรงๆ วิธีใช้งานง่ายมาก เพียงเปิด OpenCode แล้วพิมพ์คำสั่งเป็นภาษาธรรมชาติได้เลย นอกจากนี้ยังมีทางลัดสำคัญคือคำสั่ง ultrawork หรือ ulw เมื่อพิมพ์คำสั่งนี้ Sisyphus จะเข้าสู่โหมดทำงานอัตโนมัติ โดยที่เขาจะพยายามทำงานให้เสร็จเองโดยไม่ต้องถามคุณยืนยันทุกขั้นตอน เหมาะกับงานที่คุณมั่นใจว่าถ้าส่งให้เขาทำแล้วจะไม่มีอะไรผิดพลาดร้ายแรง

## สไลด์ที่ 15 — Prometheus ผู้วางแผน

Prometheus ทำหน้าที่เหมือนสถาปนิกที่ออกแบบบ้านทั้งหลังก่อนก่อสร้าง ใช้ Kimi K2.6 เป็นหลัก คุณควรใช้เมื่อเผชิญหน้ากับงานใหญ่ที่ไม่แน่ใจขอบเขต หรืองานที่ต้องการให้มีคนช่วยแบ่งเป็นขั้นตอนที่ชัดเจน ขั้นตอนการใช้งานมีห้าขั้นตอน ขั้นตอนแรก กดแป้น Tab ที่ prompt แล้วเลือก Prometheus ขั้นตอนที่สอง ตอบคำถาม Interview เพื่อกำหนดขอบเขตงาน ขั้นตอนที่สาม Prometheus จะวิเคราะห์และสร้างแผนงานที่ครอบคลุม ขั้นตอนที่สี่ คุณอนุมัติแผน และขั้นตอนสุดท้าย พิมพ์ start work เพื่อให้ Prometheus ส่งต่อแผนงานให้ Atlas ลงมือทำทันที ข้อดีของการวางแผนก่อนคือ คุณจะได้แผนที่ครอบคลุม ไม่ลืม edge case สำคัญ และรู้ตั้งแต่ต้นว่าต้องทำอะไรบ้าง

## สไลด์ที่ 16 — Hephaestus ผู้คิดลึก

ถ้างานที่คุณเจอต้องการความคิดเชิงลึก การวิเคราะห์ซับซ้อน หรือ debug ปัญหาที่ Agent ทั่วไปทำไม่สำเร็จ นั่นเป็นสัญญาณว่าคุณต้องการ Hephaestus Hephaestus เป็น Agent ระดับเชี่ยวชาญที่ใช้ GPT 5.4 เป็นหลัก ถูกออกแบบมาสำหรับ deep reasoning โดยเฉพาะ สัญญาณที่บ่งบอกว่าควรเรียกเขามาช่วยมีอยู่สามประการ ประการแรก เมื่อ Agent ตัวอื่นวนเวียนไม่หาสาเหตุเจอ ประการที่สอง เมื่อต้องค้นคว้า domain ที่ไม่คุ้นเคย เช่น library ใหม่ หรือ architecture ซับซ้อน และประการที่สาม เมื่อต้องออกแบบระบบใหม่หรือ refactor ที่กระทบหลายส่วน วิธีเรียกใช้คือกด Tab ที่ prompt แล้วเลือก Hephaestus จากนั้นอธิบายปัญหาอย่างละเอียด เขาจะใช้เวลาคิดวิเคราะห์อย่างถี่ถ้วนก่อนเสนอทางออกที่ดีที่สุด

## สไลด์ที่ 17 — ตารางตัดสินใจ

ตอนนี้เรามาดูตารางตัดสินใจกันว่าในแต่ละสถานการณ์ควรใช้ Agent ไหน ถ้าเป็นงานเล็กน้อย เช่น แก้ typo หรือแก้ไฟล์เดียว ใช้ Sisyphus ตรงๆได้เลย ถ้าเป็นฟีเจอร์ใหม่หรืองานขนาดกลาง ใช้ Prometheus วางแผนแล้วส่งให้ Atlas ทำ ถ้าเป็น refactor ระบบใหญ่ เริ่มจาก Prometheus ส่งให้ Atlas และปรึกษา Hephaestus ถ้ายังไม่รู้ว่าต้องทำอะไรดี ให้ Prometheus สัมภาษณ์คุณเพื่อช่วยกำหนดขอบเขต ถ้าติด bug ลึกหรือต้องการค้นคว้าเชิงลึก ใช้ Hephaestus โดยเฉพาะ ถ้าต้องการ review โค้ดที่เขียนเสร็จ ใช้ Code Reviewer เพื่อตรวจสอบคุณภาพและความปลอดภัย ถ้าต้องการให้ AI วิเคราะห์ภาพ วิดีโอ หรือเว็บ ใช้ Multimodal Looker และถ้าขี้เกียจอธิบายรายละเอียดมาก ใช้ ultrawork ใน Sisyphus เพราะเขาจะพยายามทำงานให้เสร็จโดยอัตโนมัติ

## สไลด์ที่ 18 — สรุป

มาสรุปสิ่งสำคัญที่ต้องจำกันอีกครั้ง ข้อหนึ่ง เลือก OpenCode เพราะเปิดกว้าง รองรับ model หลายตัว ปรับแต่งได้ลึก และมีระบบ Agent orchestration ที่เหนือกว่าคู่แข่ง ข้อสอง งานเล็กน้อย แก้ไขง่ายๆ ใช้ Sisyphus พิมพ์ตรงๆ ได้เลย ข้อสาม งานขนาดกลางใช้ Sisyphus กับ ultrawork ข้อสี่ งานใหญ่ต้องวางแผนละเอียด ใช้ Prometheus แล้วส่งให้ Atlas ทำ ข้อห้า งานที่ต้องการ deep reasoning เพิ่ม Hephaestus ข้อหก ใช้ Code Reviewer เพื่อตรวจสอบคุณภาพโค้ด ข้อเจ็ด ใช้ Multimodal Looker เพื่อวิเคราะห์ภาพ เสียง วิดีโอ และเว็บ และหลักการง่ายๆ คือเลือก Agent ให้ถูกตัว งานก็จะเสร็จไว ไม่เสียเวลา และได้ผลลัพธ์ที่ดีที่สุด ขอบคุณที่รับชมและรับฟังจนจบครับ
